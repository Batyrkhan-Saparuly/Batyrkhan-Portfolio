Game Description

A3-Dcollect-a-thon traversal game revolving around a lizardman who is attempting to scale a mountain. As you progress the game by collecting runes (or power-ups), you can access further areas which will allow you to steadily scale the mountain.

Contributions

  • Implemented Main Menu and Pause Menu, including the UI
  • Implemented basic Save/Load system
  • Implemented dynamic footstep sounds by line tracing
  • Implemented all SFX, ambience, and music

Challenges

  • Venturing into Unreal Engine and its Blueprints system marked a significant challenge due to its complex learning curve. This was my inaugural experience with both the engine and the framework. It took me a lot of time to understand how Blueprints and UE5 work in general.
  • It was my first time working with a team on such a big project, and it was quite challenging to adapt to the whole process. I faced the difficulties from the beginning, when we had to form our teams, since I was not as experienced as other Tech Artists
  • The main challenge for our team was the Version Control, none of us had enough will and time to understand Git LFS, and we ended up losing a lot of time because of that.

Version Control

Started with GitHub, transitioned to Git LFS. However, none of us could really figure out how to effectively use Git LFS and how to merge properly. So we ended up simply using Google Drive to manage and merge different versions of the game. It was not very effective, since one of us had to download all of the versions to then merge all the Blueprints, Sounds, UIs and etc to one version.

Code Architecture

Focused on the code for the main menu and pause menu, implemented fully using Unreal Engine's Blueprint system. Also worked on the UI for the widgets. Implemented the save/load functionalities, incorporated within the main and pause menus. That took the most time, since incorporating so many connected things was confusing. Most of the work for Save/Load is done in GameInstance, where I created the functions.
